## ❌ ERROR CRÍTICO ENCONTRADO

### F1-Budget Multipliers Incorrectos

**Configuración Usada (INCORRECTA):**
```bash
--f1-budget-thresholds 0.35 0.20
--f1-budget-multipliers 30 70 100  # ← ERROR FATAL
```

**Debería ser:**
```bash
--f1-budget-thresholds 0.45 0.20
--f1-budget-multipliers 0.0 0.5 1.0  # ← CORRECTO
```

### Impacto

Los multipliers son **factores** (0.0-1.0), NO cantidades absolutas:
- Correcto: `0.0` = skip, `0.5` = 50% del budget, `1.0` = 100%
- Incorrecto: `30, 70, 100` = budgets masivos que exceden límites

**Resultado:**
- Todas las clases rechazadas por `anchor_quality < 0.50`
- 0/20 seeds generaron datos sintéticos
- Mensaje: "No se generaron ejemplos sintéticos tras el filtrado"
- ~$20 USD gastados sin resultados útiles

### F1-Budget Thresholds

**This Run (0.35/0.20):**
```python
if baseline_f1 >= 0.35:        # HIGH tier
    budget_multiplier = 30
elif baseline_f1 >= 0.20:      # MID tier
    budget_multiplier = 70
else:                          # LOW tier
    budget_multiplier = 100
```

**Validated Config (0.45/0.20) - Batch 5 Phase A:**
```python
if baseline_f1 >= 0.45:        # HIGH tier
    budget_multiplier = 30
elif baseline_f1 >= 0.20:      # MID tier
    budget_multiplier = 70
else:                          # LOW tier
    budget_multiplier = 100
```

### Why Different?

**Source of 0.35 threshold:**
- TIER S experiments (Nov 4, 2025)
- Batch 1E (Nov 4, 2025)
- Based on "Skip augmentation for F1 > 0.35" recommendation

**Source of 0.45 threshold:**
- Batch 5 Phase A (Nov 12, 2025) - Presentación Tesis 3
- Achieved +1.00% ± 0.07% macro F1
- Used in presentation results

**Impact:**
- With 0.35: More classes in MID tier (0.35-0.45 range)
- With 0.45: More classes in HIGH tier (protected from augmentation)
- Expected: 0.35 might give slightly more aggressive augmentation
