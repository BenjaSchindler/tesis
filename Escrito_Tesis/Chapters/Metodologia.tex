El dataset seleccionado presenta características representativas de escenarios reales de clasificación de texto con desbalance severo. Entre estas características destacan: distribución de frecuencias altamente asimétrica, superposición semántica entre clases relacionadas, y heterogeneidad intra-clase pronunciada.

Tres características hacen del corpus MBTI un benchmark apropiado para validar técnicas de aumento sintético. \subsection{Preprocesamiento y División de Datos}
\label{subsec:preprocesamiento}

El preprocesamiento aplica transformaciones mínimas para preservar información discriminativa. Las operaciones incluyen: remoción de URLs mediante expresiones regulares, eliminación de caracteres especiales exceptuando puntuación básica, y normalización de espacios múltiples. El sistema no aplica stemming ni lematización porque estas técnicas colapsan variantes morfológicas que pueden transportar información discriminativa sobre el tipo de personalidad.

\subsection{Configuración de Embeddings Semánticos}
\label{subsec:embeddings}

La representación vectorial emplea el modelo \texttt{all-mpnet-base-v2} de SentenceTransformers, seleccionado por su balance entre calidad y eficiencia. Este modelo produce embeddings de 768 dimensiones con normalización L2 automática y alcanza aproximadamente 900 sentencias por segundo en GPU.

La validación empírica demostró que los embeddings semánticos son requisito fundamental para el aumento efectivo. En experimentos comparativos, TF-IDF con 10,000 características produjo degradación de -8.8\% tras el aumento. En contraste, los embeddings semánticos lograron una mejora de +5.9\% bajo configuración idéntica. Esta diferencia de 14.7 puntos porcentuales evidencia que la representación semántica captura relaciones que TF-IDF no puede modelar.

\subsection{Clasificador y Métricas de Evaluación}
\label{subsec:baseline_metricas}

El clasificador emplea regresión logística multinomial con estrategia uno-contra-resto. El parámetro \texttt{class\_weight='balanced'} implementa ponderación automática inversamente proporcional a las frecuencias de clase según la Ecuación \ref{eq:class_weight}.

\begin{equation}
w_c = \frac{N}{C \times N_c}
\label{eq:class_weight}
\end{equation}

En esta ecuación, $w_c$ denota el peso de la clase $c$, $N$ es el total de muestras, $C=16$ es el número de clases, y $N_c$ cuenta los ejemplos de la clase $c$. Esta ponderación compensa el desbalance durante el entrenamiento.

La métrica primaria es Macro F1-score, definida en la Ecuación \ref{eq:macro_f1}:

\begin{equation}
\text{Macro-F1} = \frac{1}{C} \sum_{c=1}^{C} F1_c = \frac{1}{C} \sum_{c=1}^{C} \frac{2 \cdot P_c \cdot R_c}{P_c + R_c}
\label{eq:macro_f1}
\end{equation}

Esta métrica asigna peso unitario a cada clase independientemente de su frecuencia. Por tanto, las clases minoritarias tienen igual importancia que las mayoritarias en la evaluación. El valor línea base de Macro F1 para este dataset es 0.2045, reflejando la dificultad inherente del problema de clasificación.

\subsection{Protocolo de Validación K-Fold}
\label{subsec:kfold}

La validación experimental emplea K-Fold estratificado con 5 particiones y 3 repeticiones, totalizando 15 folds independientes. Este protocolo proporciona robustez estadística superior a la validación con semillas múltiples sobre una única partición, ya que cada muestra aparece exactamente una vez en el conjunto de prueba por repetición.

La significancia estadística se evalúa mediante prueba t de una muestra sobre las diferencias pareadas entre el modelo línea base y el aumentado. Para cada fold $i$, se calcula $\delta_i = F1_{aumentado,i} - F1_{baseline,i}$. El estadístico t se obtiene según la Ecuación \ref{eq:ttest}:

\begin{equation}
t = \frac{\bar{\delta}}{s_{\delta} / \sqrt{n}}
\label{eq:ttest}
\end{equation}

donde $\bar{\delta}$ es la mejora promedio, $s_{\delta}$ su desviación estándar, y $n=15$ el número de folds. Esta prueba evalúa la hipótesis nula de que la mejora promedio es cero. Se reportan p-valores e intervalos de confianza al 95\%. Un resultado se considera estadísticamente significativo cuando $p < 0.05$.

\section{Arquitectura del Sistema SMOTE-LLM}
\label{sec:vision_general}

La arquitectura propuesta integra técnicas clásicas de sobremuestreo con capacidades generativas de modelos de lenguaje de gran escala (LLM). El sistema emplea controles adaptativos que responden a características locales del espacio de embeddings, ajustando dinámicamente los parámetros de generación según el contexto de cada clase.

\subsection{Pipeline de Procesamiento}
\label{subsec:arquitectura_endtoend}

El sistema ejecuta un pipeline de siete etapas secuenciales:

\begin{enumerate}
\item \textbf{Embeddings semánticos}: Codificación de textos mediante \texttt{all-mpnet-base-v2} (768 dimensiones).
\item \textbf{Validación K-Fold estratificada}: División en 5 particiones $\times$ 3 repeticiones = 15 folds.
\item \textbf{Evaluación línea base}: Entrenamiento del clasificador y cálculo de métricas por clase para cada fold.
\item \textbf{Control de calidad}: Verificación de pureza y cohesión antes de generar.
\item \textbf{Generación adaptativa}: Para cada clase objetivo ($F1 < 0.40$ o $n < 500$):
    \begin{itemize}
    \item Agrupamiento adaptativo (K-Means, $K_{max}$=18)
    \item Selección de anclas (medoide por grupo)
    \item Generación LLM (GPT-4o-mini, $\tau$=0.3)
    \item Filtrado en cascada (4 filtros)
    \end{itemize}
\item \textbf{Entrenamiento aumentado}: Combinación de muestras originales y sintéticas con peso $w$=1.0.
\item \textbf{Evaluación estadística}: Prueba t pareada e intervalos de confianza al 95\%.
\end{enumerate}

La Figura \ref{fig:pipeline_completo} ilustra el flujo de datos completo del sistema.

\subsection{Componentes Clave del Sistema}
\label{subsec:componentes_clave}

Cuatro innovaciones metodológicas diferencian la arquitectura propuesta de enfoques previos:

\begin{itemize}
\item \textbf{Generación mediante anclas}: Ejemplos representativos de cada grupo condicionan la generación del LLM, proporcionando contexto semántico específico.
\item \textbf{Validación en cascada}: Cuatro filtros secuenciales validan aspectos ortogonales de calidad antes de aceptar cada sintético.
\item \textbf{Umbrales adaptativos}: Los umbrales de aceptación varían según la pureza del ancla, siendo más estrictos en regiones de alta superposición entre clases.
\item \textbf{Ponderación uniforme}: Los sintéticos que superan el filtrado reciben peso igual a las muestras reales ($w$=1.0), contrariamente a la práctica común de asignar pesos reducidos.
\end{itemize}

Las siguientes secciones detallan cada componente con su respectiva validación experimental.

\section{Componente 1: Agrupamiento Adaptativo}
\label{sec:clustering}

\subsection{Justificación del Agrupamiento Intra-Clase}
\label{subsec:justificacion_clustering}

El agrupamiento subdivide cada clase objetivo en grupos semánticamente coherentes antes de aplicar el aumento. Esta segmentación reconoce que las categorías complejas manifiestan múltiples subclusters temáticos. Por ejemplo, un tipo de personalidad INTJ puede escribir sobre estrategia empresarial, análisis técnico, o filosofía. Generar sintéticos mediante interpolación directa entre textos de subclusters diferentes produciría contenido incoherente.

El algoritmo K-Means agrupa las muestras de cada clase en el espacio de embeddings. Se emplea la métrica de silueta para evaluar la calidad del agrupamiento. Esta métrica cuantifica cuán similar es cada muestra a su propio grupo comparado con otros grupos, con valores entre -1 y 1 donde valores mayores indican mejor separación.

\subsection{Fórmula de Grupos Dinámicos}
\label{subsec:formula_clusters}

El número de grupos $K$ se ajusta dinámicamente según el tamaño de la clase mediante la Ecuación \ref{eq:n_clusters}:

\begin{equation}
K = \max(2, \min(K_{max}, \lfloor n / 60 \rfloor))
\label{eq:n_clusters}
\end{equation}

donde $n$ representa el conteo de muestras y $K_{max}=18$ limita la fragmentación excesiva. La división por 60 garantiza un mínimo de 60 muestras por grupo, umbral empírico para la estabilidad estadística de los centroides. El límite inferior de 2 grupos asegura la detección de al menos dos subtemas distintos.

\subsection{Validación del Parámetro $K_{max}$}
\label{subsec:validacion_kmax}

Para determinar el valor óptimo de $K_{max}$, se evaluaron configuraciones desde 1 hasta 24 grupos máximos. La Tabla \ref{tab:clustering_validation} presenta los resultados de esta validación.

\input{Tables/tab_clustering_validation}

El análisis revela que $K_{max}=18$ alcanza el óptimo con una mejora de +0.40 pp en Macro F1 (p=0.003). Este resultado refleja un balance entre dos fuerzas opuestas:

\textbf{Subagrupamiento} ($K_{max} < 5$): Valores bajos limitan la capacidad de capturar heterogeneidad intra-clase. Por ejemplo, con $K_{max}=1$ (sin agrupamiento), se selecciona una única ancla para toda la clase, ignorando que los tipos de personalidad manifiestan múltiples subtemas. Los sintéticos generados tienden a concentrarse en el subtema del ancla, dejando otros subtemas sin aumento. La mejora de solo +0.13 pp confirma esta limitación.

\textbf{Sobreagrupamiento} ($K_{max} > 18$): Valores altos fragmentan excesivamente los grupos. Con $K_{max}=24$, algunos grupos contienen menos de 30 muestras, insuficientes para que el medoide sea representativo. La métrica de silueta (0.037) se mantiene pero la coherencia semántica de los grupos disminuye. La mejora de +0.22 pp (p=0.078, no significativo) confirma la degradación.

La configuración óptima $K_{max}=18$ produce grupos con 60-200 muestras cada uno, suficientes para caracterización estadística robusta. La métrica de silueta de 0.037 es modesta pero consistente con la complejidad semántica del corpus. La Figura \ref{fig:clustering_validation} visualiza estas tendencias.

\section{Componente 2: Selección de Anclas}
\label{sec:seleccion_anclas}

Las anclas son muestras representativas que condicionan la generación del LLM. La selección apropiada de anclas determina la calidad de los sintéticos generados. Esta sección describe y compara cuatro estrategias de selección.

\subsection{Estrategias de Selección}
\label{subsec:estrategias_anclas}

La selección de anclas determina qué ejemplos condicionan la generación del LLM. Una ancla apropiada debe cumplir múltiples criterios: representar fielmente el contenido semántico del grupo, residir en una región estable del espacio de embeddings, y proporcionar contexto suficientemente específico para guiar la generación. Se evaluaron cuatro estrategias con fundamentos teóricos distintos:

\begin{itemize}
\item \textbf{Medoide}: Selecciona el punto que minimiza la distancia promedio a todos los miembros del grupo, formalmente definido como $m = \arg\min_{x \in C} \sum_{y \in C} d(x, y)$ donde $C$ es el conjunto de puntos del grupo y $d$ la distancia euclidiana. A diferencia del centroide (promedio aritmético de coordenadas), el medoide es una muestra real del conjunto. Esta propiedad garantiza que el ancla sea un texto auténtico de la clase, evitando artefactos que podrían surgir de promediar representaciones vectoriales. Computacionalmente, el cálculo del medoide requiere $O(n^2)$ comparaciones por grupo, pero dado que los grupos tienen típicamente 60-200 muestras, el costo es aceptable.

\item \textbf{Filtrado por calidad}: Prioriza muestras con alta pureza local, definida como la fracción de vecinos K-NN pertenecientes a la misma clase (Ecuación \ref{eq:purity_formal}). Esta estrategia selecciona anclas en regiones del espacio semántico donde la clase es dominante, minimizando el riesgo de que el LLM genere sintéticos que se asemejen a clases vecinas. El razonamiento es que anclas en regiones ``limpias'' producirán sintéticos que hereden esa pureza. Sin embargo, esta estrategia puede sesgar la generación hacia subconjuntos atípicos de la clase que casualmente están aislados de otras categorías.

\item \textbf{Diversidad}: Maximiza la distancia mínima entre anclas seleccionadas mediante el algoritmo de muestreo del punto más lejano (\textit{farthest point sampling}). El proceso es iterativo: se selecciona primero el centroide, luego en cada paso se añade el punto más distante a todos los previamente seleccionados. Esta estrategia garantiza cobertura amplia del espacio semántico, evitando que múltiples anclas representen el mismo subtema. El riesgo es seleccionar puntos periféricos o atípicos que generen sintéticos poco representativos.

\item \textbf{Ensemble}: Combina las tres estrategias anteriores mediante una puntuación ponderada: $score(x) = \alpha \cdot centralidad(x) + \beta \cdot pureza(x) + \gamma \cdot diversidad(x)$ con pesos $\alpha=0.4$, $\beta=0.3$, $\gamma=0.3$. La hipótesis es que integrar múltiples criterios produce anclas superiores a cualquier criterio individual. Sin embargo, la combinación lineal puede diluir las fortalezas de cada componente sin eliminar sus debilidades.
\end{itemize}

\subsection{Validación de Estrategias}
\label{subsec:validacion_anclas}

La Tabla \ref{tab:anchor_strategies} presenta los resultados comparativos de las cuatro estrategias.

\input{Tables/tab_anchor_strategies}

Contrariamente a la hipótesis inicial de que el ensemble sería superior, la estrategia de \textbf{medoide} obtiene el mejor resultado con +0.34 pp de mejora (p=0.005). Este resultado aparentemente contraintuitivo tiene explicación fundamentada:

El medoide combina dos propiedades deseables que las otras estrategias sacrifican parcialmente. Primero, es un punto real del conjunto (no una construcción artificial), garantizando que el ancla sea un texto auténtico de la clase. Segundo, minimiza la distancia promedio a todos los miembros del grupo, asegurando centralidad. Esta combinación de autenticidad y representatividad produce anclas que generan sintéticos coherentes.

La estrategia de filtrado por calidad (+0.22 pp, p=0.019) sacrifica representatividad: selecciona puntos en regiones ``limpias'' que pueden ser atípicos de la clase. El ensemble (+0.25 pp, p=0.012), aunque combina criterios, diluye la fortaleza del medoide sin eliminar las debilidades de los otros componentes.

\subsection{Cálculo de Pureza Local}
\label{subsec:pureza}

La pureza local cuantifica la dominancia de una clase en el vecindario de una muestra. Para una muestra $x$ con etiqueta $y$, la pureza se define en la Ecuación \ref{eq:purity_formal}:

\begin{equation}
P(x) = \frac{1}{k} \sum_{i=1}^{k} \mathbb{1}[y_{neighbor_i(x)} = y]
\label{eq:purity_formal}
\end{equation}

donde $k=15$ vecinos y $\mathbb{1}[\cdot]$ es la función indicadora. Valores cercanos a 1.0 indican regiones donde la clase domina el vecindario. Valores bajos (típicamente 0.25-0.45 en este corpus) revelan superposición significativa con otras categorías. La pureza informa las decisiones de umbrales adaptativos descritas en la Sección \ref{sec:filtrado}.

\section{Componente 3: Generación Sintética con LLM}
\label{sec:generacion_llm}

\subsection{Selección del Modelo Generativo}
\label{subsec:seleccion_modelo}

El sistema emplea GPT-4o-mini como modelo generativo. Esta selección responde a un análisis de costo-beneficio: con tarifa de \$0.00017 por mil tokens, el costo por sintético es aproximadamente \$0.00006. Este costo permite experimentación extensiva sin restricciones presupuestarias. La calidad de generación es comparable a modelos más costosos para la tarea específica de parafraseo contextualizado.

\subsection{Arquitectura de Prompts}
\label{subsec:prompts}

El prompt estructurado incluye cinco componentes: (1) definición del rol de sistema como generador de texto, (2) descripción de la tarea de parafraseo, (3) texto del ancla principal como referencia, (4) lista de K vecinos ordenados por similaridad como contexto, y (5) instrucciones específicas de generación.

Los vecinos reciben ponderación por distancia inversa según la Ecuación \ref{eq:weight_neighbors}:

\begin{equation}
w_i = \frac{sim(\mathbf{v}_{ancla}, \mathbf{v}_i)}{\sum_{j=1}^{K} sim(\mathbf{v}_{ancla}, \mathbf{v}_j)}
\label{eq:weight_neighbors}
\end{equation}

Esta ponderación enfatiza vecinos más similares al ancla, proporcionando contexto relevante sin introducir ruido de vecinos periféricos.

\subsection{Validación de K Vecinos para Ponderación}
\label{subsec:validacion_k}

Es importante distinguir este parámetro del número de ejemplos textuales mostrados en el prompt, que se evalúa posteriormente en la Sección \ref{sec:prompting_strategies}. El valor K=200 alcanza el mejor resultado (+0.33 pp, p=0.0085), aunque la mejora marginal respecto a K=25 no justifica el costo computacional adicional en la mayoría de casos. \subsection{Validación de Temperatura}
\label{subsec:validacion_temperatura}

La temperatura $\tau$ controla la entropía de la distribución de probabilidad sobre tokens. Valores bajos producen salidas más determinísticas; valores altos aumentan la diversidad pero pueden introducir incoherencia. La distribución de probabilidad se calcula según la Ecuación \ref{eq:temperatura}:

\begin{equation}
P(t_i) = \frac{\exp(z_i / \tau)}{\sum_j \exp(z_j / \tau)}
\label{eq:temperatura}
\end{equation}

La Tabla \ref{tab:temperature_validation} presenta la validación de este parámetro.

\input{Tables/tab_temperature_validation}

La temperatura $\tau$=0.3 alcanza el mejor resultado (+0.18 pp, p=0.027). Este valor bajo produce salidas más determinísticas y coherentes. Temperaturas mayores (0.7-0.9) aumentan la diversidad léxica pero introducen mayor variabilidad semántica, lo cual degrada la tasa de aceptación en los filtros posteriores.

\subsection{Validación de Presupuesto}
\label{subsec:validacion_presupuesto}

El presupuesto $B_c$ para cada clase $c$ determina cuántos sintéticos se generan. Se calcula como porcentaje del tamaño de clase según la Ecuación \ref{eq:budget}:

\begin{equation}
B_c = \lfloor p \times N_c \times M_c \rfloor
\label{eq:budget}
\end{equation}

donde $p$ es el porcentaje base, $N_c$ el número de muestras de la clase, y $M_c$ un multiplicador adaptativo basado en la pureza promedio de la clase.

La Tabla \ref{tab:budget_validation} presenta los resultados de validación para diferentes presupuestos.

\input{Tables/tab_budget_validation}

El presupuesto de 20\% alcanza el mejor resultado (+0.48 pp, p=0.0063). Presupuestos mayores (25-30\%) no producen mejoras significativas adicionales, sugiriendo saturación del beneficio del aumento. Presupuestos menores (5-10\%) subexplotan el potencial de mejora.

\section{Componente 4: Filtrado Multi-Nivel}
\label{sec:filtrado}

El filtrado valida cada sintético antes de incorporarlo al conjunto de entrenamiento. Un filtrado riguroso es esencial para prevenir la degradación del clasificador por sintéticos de baja calidad.

\subsection{Arquitectura de Filtrado en Cascada}
\label{subsec:cascada}

El filtrado en cascada implementa el principio de defensa en profundidad: cada etapa evalúa un aspecto ortogonal de calidad, y solo los sintéticos que superan todas las etapas se incorporan al conjunto de entrenamiento. La ordenación de filtros sigue una lógica de costo computacional creciente: los filtros más baratos (longitud) preceden a los más costosos (embedding y clasificación).

El pipeline procesa cada sintético mediante cuatro etapas secuenciales. El rechazo en cualquier etapa descarta el candidato inmediatamente, evitando cómputo innecesario:

\begin{enumerate}
\item \textbf{Filtro de longitud} ($O(1)$): Verifica que el texto tenga entre 10 y 200 palabras. Textos muy cortos ($<$10 palabras) carecen de contenido discriminativo para distinguir entre tipos de personalidad. Textos muy largos ($>$200 palabras) frecuentemente contienen repeticiones o divagaciones introducidas por el LLM al intentar extender la generación. Este filtro rechaza aproximadamente el 8\% de los candidatos y es computacionalmente trivial.

\item \textbf{Filtro de similaridad} ($O(n)$): Calcula la similaridad coseno promedio entre el embedding del sintético y sus K=15 vecinos más cercanos dentro de la clase. El umbral $\theta_{sim} \geq 0.65$ verifica que el sintético resida en una región poblada del espacio semántico. Un sintético aislado (baja similaridad con vecinos) probablemente representa contenido anómalo generado por alucinación del LLM. Este filtro rechaza aproximadamente el 15\% de los candidatos restantes.

\item \textbf{Filtro de pureza K-NN} ($O(n \cdot C)$): Evalúa qué fracción de los K vecinos del sintético (considerando todas las clases) pertenecen a la clase objetivo. El umbral $\theta_{pur} \geq 0.40$ confirma coherencia semántica: un sintético cuyos vecinos son mayoritariamente de otras clases probablemente exhibe características de esas clases en lugar de la clase objetivo. Este filtro es el más selectivo, rechazando aproximadamente el 45\% de los candidatos restantes.

\item \textbf{Filtro de coherencia con ancla} ($O(1)$): Verifica similaridad coseno directa ($\theta_{anc} \geq 0.55$) entre el sintético y el ancla que condicionó su generación. Este filtro detecta casos donde el LLM ignora el contexto proporcionado y genera contenido genérico no relacionado con el ancla. Aunque debería ser redundante con los filtros anteriores, en la práctica captura aproximadamente el 12\% adicional de sintéticos problemáticos.
\end{enumerate}

La tasa de aceptación acumulada es aproximadamente 20\%, lo cual implica que se generan 5 candidatos por cada sintético finalmente incorporado. Este ratio refleja un compromiso deliberado: la generación es económica (\$0.00006/candidato), mientras que incorporar sintéticos de baja calidad degrada irreversiblemente el clasificador.

\section{Componente 5: Ponderación de Muestras Sintéticas}
\label{sec:ponderacion}

\subsection{Motivación}
\label{subsec:motivacion_ponderacion}

El enfoque tradicional en aumento de datos asigna peso reducido (típicamente 0.5) a las muestras sintéticas. La justificación es que los sintéticos son inherentemente menos confiables que las muestras reales. Sin embargo, esta suposición ignora que los sintéticos que superan filtros rigurosos pueden ser de calidad comparable a las muestras originales.

El peso asignado a cada sintético durante el entrenamiento modula su influencia en la función de pérdida. Un peso de 1.0 significa que el sintético contribuye igual que una muestra real. Un peso de 0.5 significa que contribuye la mitad.

\section{Estrategias de Prompting para Generación Sintética}
\label{sec:prompting_strategies}

Los componentes descritos en las secciones anteriores establecen la arquitectura fundamental del sistema SMOTE-LLM. La validación de K vecinos (Sección \ref{subsec:validacion_k}) optimizó el tamaño del vecindario para cálculos de ponderación y filtrado, estableciendo K=200 como valor óptimo (+0.33 pp). Sin embargo, ese parámetro determina cuántos vecinos se utilizan en los \textit{cálculos internos} del sistema, no cuántos ejemplos se muestran textualmente al LLM.

Esta sección aborda una pregunta distinta: ¿cuántos ejemplos deben incluirse \textit{explícitamente} en el prompt como demostraciones para el modelo de lenguaje? Este número de ejemplos (denominado \textit{n-shot}) afecta directamente la capacidad de aprendizaje en contexto del LLM. Los experimentos mantienen K=200 para ponderación mientras varían el número de ejemplos textuales.

\subsection{Fundamento del Aprendizaje en Contexto}
\label{subsec:fundamento_prompting}

Los modelos de lenguaje de gran escala exhiben capacidades de aprendizaje en contexto (\textit{in-context learning}): pueden adaptar su comportamiento basándose en ejemplos proporcionados en el prompt, sin actualizar sus parámetros internos. Esta propiedad emergente, documentada por Brown et al. (2020) en GPT-3, permite que el modelo aprenda patrones específicos de la tarea a partir de demostraciones.

En el contexto de generación sintética para clasificación de texto, el aprendizaje en contexto tiene implicaciones directas. Cuando el prompt incluye múltiples ejemplos de una clase específica (por ejemplo, publicaciones de usuarios INTJ), el modelo puede inferir patrones estilísticos, vocabulario característico y estructuras discursivas propias de esa categoría. La hipótesis de trabajo es que más ejemplos en contexto proporcionan información más rica, permitiendo generaciones de mayor coherencia semántica.

Las estrategias de prompting se clasifican según el número de ejemplos incluidos. El prompting sin ejemplos (zero-shot) solo proporciona instrucciones de tarea. El prompting con pocos ejemplos (few-shot, típicamente 1-5) incluye algunas demostraciones. El prompting con muchos ejemplos (many-shot, $>$5) maximiza el contexto disponible dentro del límite de tokens del modelo.

\subsection{Diseño Experimental}
\label{subsec:diseno_prompting}

Para evaluar el impacto del número de ejemplos en contexto, se diseñaron tres configuraciones experimentales manteniendo constantes todos los demás parámetros del sistema (agrupamiento $K_{max}$=18, estrategia de anclas medoide, cascada de 4 filtros, peso unitario).

La configuración de prompting sin ejemplos incluye únicamente la instrucción de tarea y el texto del ancla como referencia principal. El modelo debe inferir las características de la clase exclusivamente a partir de un único ejemplo.

La configuración de prompting con 3 ejemplos añade tres vecinos cercanos del ancla ordenados por similaridad coseno. Estos ejemplos proporcionan contexto adicional sobre la variabilidad aceptable dentro de la clase.

Las configuraciones de many-shot prompting (10, 20, 50, 100 y 200 ejemplos) exploran el límite superior del contexto. GPT-4o-mini con 128K tokens de contexto permite incluir hasta 200 ejemplos truncados a 200 caracteres cada uno ($\sim$10,000 tokens de ejemplos).

\label{subsec:resultados_prompting}

El costo adicional de incluir más ejemplos en el prompt es moderado pero justificable. Con 60 ejemplos truncados a 200 caracteres, el incremento en tokens de entrada representa aproximadamente \$0.0003 adicionales por generación. Considerando la mejora de +0.45 pp respecto a 10 ejemplos, el costo-beneficio es muy favorable.

La principal limitación es el límite de contexto del modelo. GPT-4o-mini admite 128,000 tokens, permitiendo hasta 200 ejemplos truncados sin restricciones. La degradación observada con 200 ejemplos no se debe a límites de tokens sino a saturación semántica.

\section{Combinación de Configuraciones mediante Ensembles}
\label{sec:ensembles}

Las secciones precedentes optimizaron componentes individuales del sistema, identificando la mejor estrategia de prompting con 60 ejemplos (+1.67 pp) como el punto de referencia individual. Una pregunta natural surge: ¿puede la combinación de múltiples configuraciones superar los resultados individuales? Esta sección explora estrategias de ensemble que combinan datos sintéticos de diferentes configuraciones.

\subsection{Hipótesis y Motivación}
\label{subsec:motivacion_ensembles}

La literatura de aprendizaje automático documenta extensivamente los beneficios de los ensembles. La combinación de múltiples modelos reduce varianza, mitiga sobreajuste, y frecuentemente supera cualquier modelo individual. El principio subyacente es la diversidad: modelos entrenados con diferentes configuraciones cometen errores distintos que se cancelan parcialmente al agregar.

Aplicado al aumento de datos sintéticos, la hipótesis de trabajo propone que diferentes configuraciones de generación (variando prompting, temperatura, volumen) producen conjuntos sintéticos con características complementarias. La concatenación de estos conjuntos podría proporcionar cobertura más amplia del espacio semántico de cada clase.

La diversidad de generación tiene fundamento teórico. Las configuraciones con diferente temperatura producen sintéticos con distinta dispersión en el espacio de embeddings. Las configuraciones con diferente número de ejemplos enfatizan distintos aspectos estilísticos. La combinación podría capturar facetas complementarias de cada clase.

\subsection{Metodología de Construcción de Ensembles}
\label{subsec:metodologia_ensembles}

Los ensembles se construyen mediante concatenación directa de conjuntos sintéticos. Para cada configuración participante, se ejecuta el pipeline completo de generación y filtrado. Los conjuntos resultantes se unen, aplicando opcionalmente deduplicación para eliminar sintéticos redundantes.

La deduplicación emplea umbral de similaridad coseno: dos sintéticos se consideran duplicados si su similaridad supera 0.95. Este umbral conservador preserva variantes léxicas mientras elimina copias casi idénticas que podrían surgir cuando diferentes configuraciones seleccionan anclas similares.

Se evaluaron cuatro estrategias de ensemble:

\begin{itemize}
\item \textbf{Ensemble de mejores configuraciones}: Combina los ganadores de cada categoría experimental (prompting 60 ejemplos, temperatura 0.3, sin filtrado). Tres configuraciones, aproximadamente 1,500 sintéticos combinados.

\item \textbf{Ensemble Top-3}: Las tres mejores configuraciones individuales por mejora en F1. Similar composición al anterior pero selección basada únicamente en rendimiento.

\item \textbf{Ensemble enfocado en clases problema}: Configuraciones diseñadas específicamente para mejorar ESFJ, ESFP y ESTJ. Multiplicadores de presupuesto aumentados para estas clases.

\item \textbf{Ensemble extendido Top-5}: Las cinco mejores configuraciones individuales. Mayor volumen pero potencial redundancia.
\end{itemize}

\subsection{Resultados de Ensembles}
\label{subsec:resultados_ensembles}

Los ensembles combinan los mejores resultados de tres categorías experimentales: (1) estrategias de prompting (mejor: 60 ejemplos, +1.67 pp), (2) configuraciones de temperatura con contexto óptimo (mejor: $\tau$=0.3, +1.18 pp), y (3) experimentos de filtrado (mejor: sin filtrado, +1.03 pp). La Tabla \ref{tab:ensemble_results} presenta los resultados de las estrategias de ensemble comparadas con la mejor configuración individual.

\input{Tables/tab_ensemble_results}

Contrariamente a la hipótesis inicial, ningún ensemble supera la mejor configuración individual. El ensemble de mejores configuraciones alcanza +0.90 pp (p = 0.004), 0.77 pp por debajo del prompting con 60 ejemplos (+1.67 pp). El ensemble Top-3 obtiene +0.89 pp (p = 0.0002), confirmando la tendencia.

Este resultado aparentemente paradójico tiene explicación fundamentada. A diferencia de los ensembles de modelos (donde diferentes algoritmos capturan patrones complementarios), los ensembles de datos sintéticos adolecen de redundancia estructural.

Primero, todas las configuraciones emplean el mismo modelo de lenguaje (GPT-4o-mini), compartiendo sesgos generativos intrínsecos. Segundo, las anclas frecuentemente coinciden: las mejores anclas (medoides de grupos densos) son seleccionadas por todas las configuraciones. Tercero, los filtros aplican los mismos criterios de aceptación, convergiendo hacia características similares en los sintéticos aceptados.

La combinación de datos sintéticos redundantes diluye las señales discriminativas. Cuando un clasificador entrena con sintéticos de alta calidad mezclados con sintéticos de calidad inferior, el promedio perjudica. La configuración individual óptima produce un conjunto curado de sintéticos, mientras que el ensemble introduce ruido de configuraciones subóptimas.

\subsection{Discusión: Limitaciones de los Ensembles de Datos Sintéticos}
\label{subsec:limitaciones_ensembles}

Los resultados contradicen la intuición de que ``más datos siempre es mejor''. En el contexto específico de aumento sintético con LLMs, la calidad prevalece sobre la cantidad. Tres factores explican esta observación.

Primero, los filtros de cascada ya optimizan por calidad. Un sintético que supera cuatro filtros secuenciales representa el 20\% superior de candidatos generados. Combinar estos sintéticos de élite con sintéticos de configuraciones menos optimizadas degrada el promedio.

Segundo, la redundancia semántica no aporta información nueva. Diferentes configuraciones que seleccionan anclas similares generan sintéticos en regiones superpuestas del espacio de embeddings. El clasificador no obtiene señales discriminativas adicionales de variantes léxicas del mismo concepto.

Tercero, el desbalance persiste. Aunque los ensembles aumentan el volumen total, las proporciones entre clases se mantienen aproximadamente constantes. Las clases mayoritarias reciben proporcionalmente más sintéticos, perpetuando el desbalance original.

La conclusión práctica es que la optimización de configuración individual supera a las estrategias de combinación. Los recursos computacionales invertidos en generar múltiples conjuntos sintéticos producen mayor retorno cuando se dedican a optimizar una única configuración de alta calidad.

\section{Optimización para Clases de Baja Frecuencia}
\label{sec:clases_raras}

Las secciones anteriores demostraron que el sistema SMOTE-LLM logra mejoras significativas en Macro F1, alcanzando hasta +1.67 pp con la configuración óptima de prompting (60 ejemplos). No obstante, el análisis de impacto por nivel (Sección \ref{subsec:impacto_tier}) reveló que este beneficio no es uniforme: las clases de baja frecuencia (ESFJ, ESFP, ESTJ con menos de 50 muestras) exhiben sistemáticamente 0\% de mejora.

Esta sección investiga estrategias específicas para resolver estas clases problemáticas, incluyendo sobremuestreo intensivo y clasificadores alternativos.

\subsection{Caracterización del Problema}
\label{subsec:caracterizacion_problema}

Tres clases del corpus MBTI exhiben comportamiento refractario al aumento sintético:

\begin{itemize}
\item \textbf{ESFJ} (42 muestras, 0.5\% del corpus): Tipo de personalidad caracterizado por extraversión, sensación, sentimiento y juicio. A pesar de generar hasta 700 sintéticos mediante sobremuestreo intensivo, el F1 por clase permanece en 0.00 con Regresión Logística.

\item \textbf{ESFP} (48 muestras, 0.6\% del corpus): Caracterizado por extraversión, sensación, sentimiento y percepción. Similar al anterior, con 944 sintéticos generados y ninguna mejora detectable.

\item \textbf{ESTJ} (39 muestras, 0.4\% del corpus): La clase más escasa, caracterizada por extraversión, sensación, pensamiento y juicio. Los sintéticos generados no producen mejora mensurable.
\end{itemize}

El denominador común es la combinación de escasez extrema (menos del 1\% cada una) con superposición semántica significativa. Estas clases comparten tres de cuatro dimensiones con múltiples clases vecinas, generando confusión sistemática en el espacio de embeddings.

La hipótesis inicial atribuía el problema a insuficiencia de datos sintéticos. Si el clasificador no puede aprender patrones discriminativos de 40 ejemplos originales, quizás necesita más sintéticos. Esta hipótesis motivó los experimentos de sobremuestreo intensivo.

\subsection{Estrategias de Sobremuestreo Intensivo}
\label{subsec:sobremuestreo_intensivo}

Se implementó un multiplicador de presupuesto de 20$\times$ para las tres clases problemáticas. Este multiplicador genera aproximadamente 800-1,000 sintéticos por clase, transformando las proporciones de 0.5\% a aproximadamente 8\% del conjunto de entrenamiento aumentado.

Los resultados con Regresión Logística (clasificador estándar del sistema) fueron parcialmente alentadores:

\begin{itemize}
\item ESFJ: +0.80 pp de mejora (única clase con respuesta positiva)
\item ESFP: 0.00 pp (sin cambio)
\item ESTJ: 0.00 pp (sin cambio)
\end{itemize}

El sobremuestreo intensivo resolvió parcialmente una de tres clases. ESFJ respondió al aumento masivo, sugiriendo que el problema original era efectivamente escasez de datos. Sin embargo, ESFP y ESTJ permanecieron impermeables, indicando una limitación más fundamental.

Esta observación condujo a una nueva hipótesis: quizás el problema no reside en los datos sino en el clasificador. La Regresión Logística asume separabilidad lineal en el espacio de características. Si las clases problemáticas requieren fronteras de decisión no lineales, ningún aumento de datos lineal resolverá el problema.

\subsection{Hipótesis del Clasificador}
\label{subsec:hipotesis_clasificador}

Los embeddings semánticos de 768 dimensiones representan un espacio de alta dimensionalidad. La Regresión Logística busca hiperplanos separadores en este espacio. Si las clases problemáticas están distribuidas en regiones que requieren fronteras curvas o combinaciones no lineales de características, el clasificador lineal fallará sistemáticamente.

Las redes neuronales multicapa (MLP) pueden aprender fronteras de decisión arbitrariamente complejas mediante composición de transformaciones no lineales. Un MLP con suficientes capas ocultas puede aproximar cualquier función de clasificación, incluyendo aquellas requeridas para separar clases semánticamente superpuestas.

Adicionalmente, los modelos basados en árboles (XGBoost, LightGBM) ofrecen capacidad no lineal mediante particiones recursivas del espacio de características. Estos modelos son frecuentemente exitosos en problemas tabulares con características de alta dimensionalidad.

\subsection{Evaluación Multi-Clasificador}
\label{subsec:evaluacion_multiclasificador}

Se evaluaron cinco clasificadores sobre el mismo conjunto de datos sintéticos (sobremuestreo 20$\times$ para clases problemáticas):

\begin{itemize}
\item \textbf{Regresión Logística}: Clasificador lineal de referencia (línea base del sistema).
\item \textbf{MLP Profundo (512-256-128)}: Perceptrón multicapa con tres capas ocultas de 512, 256 y 128 neuronas respectivamente. Activación ReLU, dropout 0.3 por capa.
\item \textbf{MLP Medio (256-128)}: Arquitectura más compacta con dos capas ocultas.
\item \textbf{XGBoost}: Gradient boosting con 200 árboles, profundidad máxima 8, tasa de aprendizaje 0.1.
\item \textbf{LightGBM}: Gradient boosting optimizado para velocidad, 200 iteraciones, 31 hojas máximo.
\end{itemize}

La Tabla \ref{tab:multiclassifier} presenta los resultados comparativos.

\input{Tables/tab_multiclassifier}

Los resultados son contundentes. El MLP Profundo alcanza +4.17 pp de mejora en Macro F1 (p = 0.0010), superando dramáticamente a todos los demás clasificadores. La Regresión Logística logra apenas +0.36 pp (no significativo). Los modelos basados en árboles no solo fallan en mejorar, sino que empeoran: XGBoost obtiene -0.43 pp y LightGBM -0.10 pp.

El contraste entre MLP y árboles requiere explicación. Los modelos de gradient boosting operan mediante particiones axis-aligned del espacio de características. Con 768 dimensiones, las particiones necesarias para separar clases semánticamente superpuestas requerirían profundidad prohibitiva. Además, estos modelos muestran línea base significativamente inferior (0.1788 para XGBoost vs 0.2272 para Regresión Logística), indicando que la arquitectura es fundamentalmente inadecuada para embeddings densos.

El MLP, en cambio, proyecta los embeddings a través de transformaciones aprendidas que pueden alinear las representaciones para facilitar la clasificación. Las capas ocultas actúan como extractores de características que reorganizan el espacio de embeddings de manera óptima para la tarea.

\subsection{Análisis: Superioridad de las Redes Neuronales}
\label{subsec:analisis_mlp}

Tres factores explican la superioridad del MLP Profundo sobre alternativas:

\textbf{Capacidad de representación}: Las tres capas ocultas (512→256→128) transforman progresivamente los embeddings de 768D hacia representaciones más compactas y discriminativas. Cada capa puede aprender combinaciones no lineales de características que separan clases superpuestas.

\textbf{Regularización efectiva}: El dropout de 0.3 por capa previene sobreajuste a los sintéticos. A diferencia de la Regresión Logística (propensa a sobreajustar en alta dimensionalidad), el MLP con dropout generaliza mejor a nuevas muestras.

\textbf{Explotación de estructura latente}: Los embeddings semánticos codifican relaciones complejas entre palabras y conceptos. El MLP puede aprender a explotar esta estructura para distinguir clases que comparten vocabulario superficial pero difieren en patrones sutiles.

La arquitectura específica (512-256-128) fue seleccionada mediante búsqueda de hiperparámetros. Arquitecturas más profundas (4+ capas) no mostraron mejoras adicionales. Arquitecturas más anchas (1024+) introdujeron sobreajuste. La configuración seleccionada representa un punto óptimo para el tamaño del dataset aumentado.

\subsection{Discusión: Limitaciones del Enfoque Basado en Texto}
\label{subsec:limitaciones_texto}

El caso de ESFP merece análisis específico. Con 48 muestras originales (más que ESFJ con 42), no es la clase más escasa. Los sintéticos generados superan los filtros de calidad con tasas comparables a otras clases. El MLP Profundo funciona correctamente para ESFJ y ESTJ. Sin embargo, ESFP permanece irresoluble.

Dos explicaciones complementarias emergen:

\textbf{Ambigüedad definitoria}: El tipo ESFP (Extravertido-Sensación-Sentimiento-Percepción) describe un perfil de personalidad orientado a la experiencia inmediata y la conexión emocional. En texto, estas características pueden manifestarse de formas muy similares a otros tipos extravertidos (ESFJ, ENFP, ESTP). La superposición semántica podría ser inherentemente alta, haciendo la distinción textual intrínsecamente difícil.

\textbf{Heterogeneidad intra-clase}: Las 48 muestras de ESFP podrían representar múltiples subclusters semánticos sin núcleo común claro. Los sintéticos generados amplifican esta heterogeneidad en lugar de consolidar un patrón discriminativo.

Estas limitaciones sugieren que ciertos problemas de clasificación requieren información más allá del texto. Características paralingüísticas (patrones de puntuación, uso de emoticonos, longitud de oraciones) o metadatos (hora de publicación, frecuencia de posteo) podrían proporcionar señales adicionales necesarias para distinguir ESFP de clases confusoras.

\subsection{Recomendaciones Prácticas}
\label{subsec:recomendaciones_practicas}

Los hallazgos de esta sección conducen a recomendaciones específicas para problemas similares:

\begin{enumerate}
\item \textbf{Priorizar optimización de clasificador}: Antes de invertir recursos en generación masiva de sintéticos, evaluar si un clasificador más potente puede explotar mejor los datos existentes. El cambio de Regresión Logística a MLP Profundo produjo mejoras mayores que cualquier estrategia de generación.

\item \textbf{Aplicar sobremuestreo selectivo}: Concentrar recursos de generación en clases específicas que muestren respuesta positiva. El multiplicador 20$\times$ para clases problemáticas es más efectivo que aumentar uniformemente todas las clases.

\item \textbf{Reconocer limitaciones}: Ciertas clases pueden ser intrínsecamente difíciles de distinguir mediante texto. En estos casos, explorar fuentes de información complementarias en lugar de persistir con estrategias que demuestran rendimientos decrecientes.

\item \textbf{Validar interacciones}: La combinación óptima (MLP + sobremuestreo intensivo) no es simplemente aditiva. Evaluar interacciones entre componentes del sistema antes de concluir optimización.
\end{enumerate}

\section{Consideraciones de Implementación}
\label{sec:consideraciones}

El sistema requiere Python 3.10+ con las siguientes dependencias principales: \texttt{scikit-learn} 1.3.0 para clasificación y agrupamiento, \texttt{sentence-transformers} 2.2.2 para embeddings semánticos, y \texttt{openai} 1.0.0 para acceso a la API de generación.

El costo de API por experimento completo es aproximadamente \$0.13, totalizando \$35 para los 273 experimentos de validación. El tiempo de ejecución por fold es aproximadamente 15 minutos en CPU estándar. Estos costos moderados demuestran la viabilidad económica del enfoque para investigación académica.
